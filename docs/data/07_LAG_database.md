# LAG database

## Overview
The LAG (Large-scale Attention based Glaucoma) database is a collection of color retinal fundus images established for glaucoma detection and for capturing ophthalmologist attention during diagnosis. The database contains **5,824** fundus images, with **2,392** positive (glaucoma) and **3,432** negative samples. All images were obtained from Beijing Tongren Hospital and the dataset creation was conducted according to the tenets of the Helsinki Declaration; as the images are fully anonymized and the study is retrospective, the authors report exemption by the medical ethics committee for informing patients. Each image is diagnosed by qualified glaucoma specialists using both morphologic and functional analyses (including intra-ocular pressure, visual field loss and manual optic disc assessment); these binary labels (positive / negative) are treated as the gold standard. The dataset publication and description are provided in the accompanying paper. 

## Images
- **Modality** -- Cropped Color retinal fundus photographs (macula/optic-disc field fundus images).
- **Native resolution** -- Fundus images in the attention-capture experiment are **500 × 500** pixels (as used in the attention experiment).
- **Attention / fixation capture** -- Attention regions were captured via a simulated eye-tracking method where ophthalmologists cleared circular regions (radius **40 pixels**) on initially blurred 500×500 images; the order and centers of cleared circles encode fixations used to generate GT attention maps (see Metadata).
- **Image format / file encoding** -- **JPG** Encoding
## Metadata
There is no central CSV manifest provided in the supplied materials. Metadata present in the dataset release is limited to per-image labels (text files) and attention masks derived from ophthalmologist fixations. Specifically:
- No **CSV** metadata files (e.g., train.csv/test.csv) are provided in the supplied description.
- Per-image `label` files exist (plain text) containing `0` or `1` values corresponding to negative / positive glaucoma; the prompt notes these are present but effectively redundant since they encode only `0`/`1` and the dataset is already split into non glucoma and suspicious glucoma by file structure.
- Attention masks (attention maps / visualization maps) are included and indicate regions of ophthalmologist attention (derived from the fixation experiment described in the Overview).
- Are generated from ophthalmologist fixations (center coordinates of cleared circles during the mouse-based attention experiment).
- Are produced by convolving all fixations with a 2D Gaussian (standard deviation set to **25**) and are normalized to values in `[0,1]`.
- Are available at a spatial resolution of **112 × 112** in the modelling pipeline (as described)

## Storage in database

### Tables populated

- **`datasets`** — One row registered with `dataset_name="LAG"`, `source_url`, `license="Unknown"`, and `modality_types=["fundus"]`.
- **`images`** — One row per image. Fields populated via `get_image_metadata_dict` plus `modality="fundus"`. The `original_image_id` is set to `"{folder_type}_{image_stem}"` (e.g., `"suspicious_glaucoma_00001"`), where `folder_type` is either `"suspicious_glaucoma"` or `"non_glaucoma"`. No laterality is stored.
- **`classification_annotations`** — One row per image. Binary glaucoma classification with `class_name="glaucoma"`, `task_type="binary"`, and `class_labels={0: "non_glaucoma", 1: "suspicious_glaucoma"}`. `class_value` is `True` for images from `suspicious_glaucoma/` and `False` for `non_glaucoma/`. Both `raw_data_id` and `provenance_chain_id` are explicitly set to `None` (folder structure is the implicit annotation source). `annotation_method="manual"`.
- **`segmentation_annotations`** — One row per image for which an attention map exists at `{folder_type}/attention_map/{image_stem}.jpg`. Stored as `annotation_type="attention_map"` using `process_segmentation_from_soft_map`. The attention maps are 112×112 grayscale JPEG soft maps generated by the AG-CNN model. `annotation_method="automatic"` (model-generated). `raw_data_id` and `provenance_chain_id` are `None`. Images without an attention map file log a warning but are not failed.
- **`dataset_splits`** / **`image_split`** — A single explicit `train` split is created and all images are assigned to it. No train/test partition is provided by the dataset.

### Provenance / raw annotation files

No files are registered in `raw_annotation_files`. The folder structure serves as the annotation source. The per-image label `.txt` files (`{folder_type}/label/{image_stem}.txt`) are read to verify the label matches the folder-implied value, but they are not registered for provenance. Classification and segmentation annotations carry `raw_data_id=None` and `provenance_chain_id=None`.

### Label verification

For each image, the script reads the corresponding `.txt` label file and checks that the integer value (0 or 1) matches the folder-implied label. A mismatch logs a warning but does not skip or fail the image. Missing label files also log a warning only.

### Special processing

Images from `suspicious_glaucoma/image/` and `non_glaucoma/image/` are processed sequentially (not concurrently). Segmentations are individually upserted. The script is idempotent: re-running does not create duplicates due to deterministic UUID generation and upsert logic.
