# DR1-2

## Overview
The DR1-2 dataset is a diabetic retinopathy image database containing fundus images organized by lesion types and referral status. The dataset consists of two main components: DR1 images stored in TIFF format and DR2 images stored in JPEG format. Images are organized by lesion categories including normal images, hard exudates, drusen, red lesions, cotton-wool spots, deep hemorrhages, and superficial hemorrhages. The dataset includes lesion-level annotations with bounding box coordinates stored in text files.

## Images
- DR1 images are stored in **TIFF** format (uppercase `.TIF` extension).
- DR2 images are stored in **JPEG** format (uppercase `.JPG` extension).
- DR1 image filenames follow patterns: `IMG####`.TIF, `IMG####` (##).TIF, or `A (###`).TIF/`c (###`).TIF (e.g., `IMG0001.TIF`, `IMG0001 (8).TIF`, `A (134).TIF`).
- DR2 image filenames follow the pattern `IM#####`.JPG (e.g., `IM000160.JPG`, `IM001010.JPG`).
- Images are organized into lesion-specific directories: Normal Images, Hard Exudates, Drusen, Red Lesions, Cotton-wool Spots, Deep Hemorrhages, and Superficial Hemorrhages.
- DR2 images are also organized by referral status: positive (referable) and negative (non-referable).
## Metadata
The metadata for the dataset is stored in text files (`.txt`) located in the **markings** directory. These files contain lesion-level annotations with bounding box coordinates in tab-separated format.

### Marking Files (.txt)
Each marking file corresponds to an annotated image and contains tab-separated lesion annotations. The format for each line is:
- **Lesion ID** -- Sequential identifier for each lesion in the image (integer, starting from 1).
- **Lesion Type** -- Type of lesion (Spanish/Portuguese terminology):
- `drusas-maculares` -- Macular drusen
- `mancha-algodonosa` -- Cotton-wool spots
- `hemorragia-profunda` -- Deep hemorrhages
- `hemorragia-superficial` -- Superficial hemorrhages
- `exsudato-duro` -- Hard exudates
- `dr-seroso-macular` -- Macular serous detachment (DR-related)
- **xmin** -- Minimum x-coordinate of bounding box (integer, pixel coordinates).
- **xmax** -- Maximum x-coordinate of bounding box (integer, pixel coordinates).
- **ymin** -- Minimum y-coordinate of bounding box (integer, pixel coordinates).
- **ymax** -- Maximum y-coordinate of bounding box (integer, pixel coordinates).
Each line represents one lesion annotation with its bounding box coordinates. The marking files contain 114 annotation files covering various lesion types.

## Splits
The dataset does not provide explicit train--test splits. Images are organized by lesion types and referral status, and any data partitioning must be performed externally.

## Storage in database

### Tables populated

- **`datasets`**: One record for DR1-2 (name, source_url=`https://www.kaggle.com/datasets/andrewmvd/diabetic-retinopathy-detection`, license=`Unknown`, `modality_types=['fundus']`).
- **`images`**: One row per image file found across all processed folders. `original_image_id` is set to the file stem (no extension). `modality='fundus'`. Image metadata (dimensions, format, etc.) is extracted via `get_image_metadata_dict`. Image UUIDs are generated from the dataset UUID and the file stem.
- **`classification_annotations`**: Two types of classification, depending on the source folder:
  - **Lesion type** (`class_name="lesion_type"`, `task_type="multi_class"`): stored for images in the root lesion-category folders. Folder-to-class mapping: `"Cotton-wool Spots"→"cotton_wool_spots"`, `"Deep Hemorrhages"→"deep_hemorrhages"`, `"Drusen"→"drusen"`, `"Hard Exudates"→"hard_exudates"`, `"Normal Images"→"normal"`, `"Red Lesions"→"red_lesions"`, `"Superficial Hemorrhages"→"superficial_hemorrhages"`. `class_labels=None` (no integer index), `annotation_method="manual"`.
  - **Referral** (`class_name="referral"`, `task_type="multi_class"`): stored for images in `DR2-images-by-referral/`. Folder-to-class mapping: `"Referable"→"referable"`, `"Non-Referable"→"non_referable"`. `class_labels=None`, `annotation_method="manual"`.
  - Images in `DR1-additional-marked-images/` are ingested as images **only** — no classification is stored for them.
- **`raw_annotation_files`**: Registered automatically per directory walk via `process_folder_tree` (one raw file / provenance chain per directory traversal call). No explicit CSV files are registered.
- **`provenance_chains`**: Created per `process_folder_tree` call, one per folder tree traversal.

### Annotation types
- **`classification_annotations`**: `class_name="lesion_type"` (multi-class, 7 classes) for DR1/DR2 lesion-category images; `class_name="referral"` (multi-class, 2 classes: `referable`/`non_referable`) for DR2 referral images. No `expert_annotation_id` or `raw_data_id` is associated.

### Splits created
- None. The dataset does not define any explicit train/test splits and no split registration is performed by the ingestion script.

### Provenance / raw annotation files registered
- Folder-tree traversal raw file entries are created per `process_folder_tree` call (one per folder tree scan), but no named annotation CSV or text files are registered. The `.txt` marking files in `markings/` are not processed or registered.

### Special processing
- The script uses `process_folder_tree` to walk all recognized directories simultaneously with `recursive=True`, determining the annotation type from the top-level folder name (`rel_path.parts[0]`).
- Files found directly at the root level (depth=0) are skipped with an error.
- Unrecognized folder names produce a warning and an error record; the image is not upserted.
- Supported image extensions: `.jpg`, `.JPG`, `.jpeg`, `.JPEG`, `.tif`, `.TIF`.
- The bounding-box annotation files in `markings/` are **not** ingested — no localization or bounding-box annotations are stored.
- The ingestion process is idempotent. Re-running it will not create duplicate images or annotations due to deterministic UUID generation and upsert logic.
