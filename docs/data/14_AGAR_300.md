# AGAR 300

## Overview
The AGAR300 dataset is a diabetic retinopathy fundus image dataset released via IEEE DataPort. It is intended to facilitate benchmarking of automated microaneurysm detection algorithms using digital retinal fundus images. The currently released database consists of 28 color fundus images that show signs of microaneurysms. The dataset is freely available for research use with proper citation.

## Images
- Modality: Color retinal fundus photography.
- Clinical focus: Diabetic Retinopathy with emphasis on Microaneurysms.
- Image format: **JPEG** (`*.jpeg`).
- Number of images: 28 color fundus images (first released set).
- Field of view: 45-degree field-of-view.
- Resolution: 2448 × 3264 pixels.
- Imaging device: Fundus photography machine (specific model not provided).
- Image naming convention: Not provided.
## Metadata
No metadata files are provided with this dataset. The dataset consists only of image files.

## Splits
No explicit training, testing, or validation splits are provided. All images are released as a single collection, and any data partitioning must be performed externally by the user.

## File Schema
```
AGAR300/
img/
```

## Storage in database

### Tables populated
- **`datasets`**: One record for AGAR300 with `modality_types=["fundus"]`. No `task_types` are set (images only, no annotations).
- **`images`**: One row per image file found in the `img/` directory, discovered via `find_images`. Fields extracted via `get_image_metadata_dict` (file path, resolution, file size, format). `modality="fundus"`. `original_image_id` is the filename stem. No `acquisition_date`, `image_quality`, or `comorbidities` set.
- **`dataset_splits`** / **`image_splits`**: A single explicit `train` split is created via `register_standard_splits`. All discovered images are assigned to this split.

### Image metadata extraction
Images are discovered recursively in `img/`. `get_image_metadata_dict` reads each file for resolution, file size, and format. No laterality detection, no patient records, and no annotation files are processed.

### Annotation types
None. This dataset contains images only; no annotations are stored.

### Splits
All images are assigned to a single `train` split (split type `"explicit"`). No test or validation split is created.

### Provenance
No raw annotation files are registered, as no annotation source files exist.

### Special processing
None. Ingestion is idempotent due to deterministic UUID generation and upsert logic.
