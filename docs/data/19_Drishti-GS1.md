# Drishti-GS1

## Overview
The Drishti-GS1 dataset is designed for optic disc and optic cup segmentation with the objective of glaucoma assessment. The dataset consists of 101 retinal fundus images collected at Aravind Eye Hospital, Madurai, India, with patient consent. 
All images are annotated by four eye experts with varying clinical experience (3, 5, 9, and 20 years). The dataset provides multiple forms of ground truth, including soft segmentation maps, average optic disc and cup boundaries, cup-to-disc ratio (CDR) values from each expert, image-level glaucoma decisions, and notching assessments. Ground truth annotations are publicly provided for the training set, while evaluation of test images requires registration.

## Images
- The dataset contains **101 retinal fundus images**.
- Images are stored in **PNG** format using an uncompressed representation.
- Image resolution is **2896 × 1944 pixels**.
- Images are acquired with a **30-degree field of view (FOV)**.
- All images are centered on the **Optic Disc (OD)**.
- Image acquisition was performed at Aravind Eye Hospital, Madurai.
- Camera model details beyond the provided information are not specified.
## Metadata
Metadata and clinical annotations are provided through Excel files and structured text files associated with each image. There is no centralized CSV file; metadata is distributed across multiple files corresponding to different annotation components.

### Metadata Files

#### Drishti-GS1_diagnosis.xlsx
This Excel file contains clinical diagnosis information for images.
- **Image identifier** -- Identifier corresponding to each fundus image.
- **Image-level decision** -- Binary classification indicating normal or glaucomatous status, derived from majority opinion (3 out of 4 experts).
#### notching_image_decisions.xls
This Excel file contains notching assessments and image-level decisions.
- **Superior notching** -- Presence or absence of notching in the superior sector.
- **Inferior notching** -- Presence or absence of notching in the inferior sector.
- **Nasal notching** -- Presence or absence of notching in the nasal sector.
- **Temporal notching** -- Presence or absence of notching in the temporal sector.
- **Image-level decision** -- Normal or glaucomatous label assessed by a single expert.
#### Notching_&_Image__level_decisions.xlsx
This Excel file, located in the `Training/GT` directory, contains additional notching and image-level decision information corresponding to the training images.

#### CDR Value Files
Files named `drishtiGS_XXX_cdrValues.txt` provide cup-to-disc ratio values.
- **Value format** -- Space-separated numeric values.
- **Number of values** -- Four values per file.
- **Meaning** -- One CDR value per expert annotation.
#### Disk Center Files
Files named `drishtiGS_XXX_diskCenter.txt` store optic disc center coordinates.
- **Format** -- Single line with space-separated `x y` pixel coordinates.
- **Meaning** -- Center of the optic disc in the image.
#### Average Boundary Files
Average boundary annotations are stored in the `AvgBoundary/` directory.
- **drishtiGS_XXX_ODAvgBoundary.txt** -- Average optic disc boundary coordinates.
- **drishtiGS_XXX_CupAvgBoundary.txt** -- Average optic cup boundary coordinates.
- **Format** -- Space-separated `x y` coordinate pairs, one per line.
- **Description** -- Dense pixel-level boundary annotations obtained by averaging expert markings over angular sectors.
#### Soft Segmentation Maps
Soft segmentation maps are stored in the `SoftMap/` directory.
- **drishtiGS_XXX_ODsegSoftmap.png** -- Soft segmentation map for the optic disc.
- **drishtiGS_XXX_cupsegSoftmap.png** -- Soft segmentation map for the optic cup.
- **Description** -- Probability maps generated by fusing annotations from all four experts.
## Splits
Explicit dataset splits are provided.
- **Training set**: 50 images, with complete ground truth annotations.
- **Test set**: 51 images.
- No validation split is provided.
- Ground truth for test images requires registration for evaluation.

## Storage in database

### Tables populated
- **`datasets`**: One record for Drishti-GS1 with `modality_types=["fundus"]`.
- **`experts`**: Four records — "Drishti-GS1 Expert 1" through "Drishti-GS1 Expert 4", each with `expertise_area` unset (defaults to `None` in the model).
- **`images`**: One row per PNG image in `Training/Images/` and `Test/Images/`. `original_image_id` is the image stem (e.g., `drishtiGS_002`). Fields extracted via `get_image_metadata_dict` and `extract_image_metadata` (for width/height required by contour processing). `modality="fundus"`. Images without a corresponding GT folder are skipped.
- **`segmentation_annotations`** (soft maps): Up to two per image — `annotation_type="optic_disc"` from `{id}_ODsegSoftmap.png` and `annotation_type="optic_cup"` from `{id}_cupsegSoftmap.png`. Processed via `process_segmentation_from_soft_map`. `expert_annotation_id=None` (consensus). PNG files are registered with `file_type=None`.
- **`segmentation_annotations`** (average boundaries): Up to two per image — `annotation_type="optic_disc"` from `{id}_ODAvgBoundary.txt` and `annotation_type="optic_cup"` from `{id}_CupAvgBoundary.txt`. Processed via `process_segmentation_from_contour` with `coordinate_format="line_separated"`. `expert_annotation_id=None` (consensus). These have different `raw_data_id` values from the soft maps, yielding distinct UUIDs.
- **`expert_annotations`**: One record per expert per image (up to 4 per image), only where the expert's marking is non-null. `annotation_task="classification"`. `annotation_value` stores `{"glaucoma_marking": -1|1, "glaucoma_label": "normal"|"glaucoma", "image_id": ..., "image_stem": ...}`. Raw data linked to `Drishti-GS1_diagnosis.xlsx`.
- **`classification_annotations`** (per-expert): One binary classification per expert per image (where marking is non-null). `task_type="binary"`, `class_name="glaucoma"`, `class_value=True` (glaucoma) or `False` (normal). Linked to the corresponding `expert_annotation` record.
- **`classification_annotations`** (consensus): One binary classification per image where the Excel's "Total" consensus column is non-null. `task_type="binary"`, `class_name="glaucoma"`, `class_value=True` (glaucoma) or `False` (normal). `expert_annotation_id=None`.
- **`localization_annotations`**: One record per image where `{id}_diskCenter.txt` exists. `structure_name="optic_disc_center"`. Processed via `process_localization_from_text_keypoint`. Linked to the disc center text file.
- **`raw_annotation_files`**:
  - `Drishti-GS1_diagnosis.xlsx` (via `process_excel`, `unified_annotation_type="classification"`).
  - `{id}_cdrValues.txt` — registered with `unified_annotation_type="segmentation"`, `file_type="txt"` (provenance only; CDR values are not stored as annotations).
  - `{id}_ODsegSoftmap.png` and `{id}_cupsegSoftmap.png` — registered with `file_type=None`.
  - `{id}_ODAvgBoundary.txt` and `{id}_CupAvgBoundary.txt` — registered with `file_type="txt"`.
  - `{id}_diskCenter.txt` — registered with `unified_annotation_type="localization"`, `file_type="txt"`.
- **`dataset_splits`** / **`image_splits`**: Explicit `train` and `test` splits. Training images → train split; test images → test split.

### Image metadata extraction
`extract_image_metadata` extracts width and height for contour-to-polygon conversion. `get_image_metadata_dict` populates standard image fields. No laterality detection and no patient records are created.

### Annotation types
- **Segmentation** (`optic_disc`, `optic_cup`): Two representations per image — soft map (intensity-based consensus) and average boundary (contour-based consensus). Each yields a distinct `segmentation_annotation` with a different `raw_data_id`.
- **Classification** (`glaucoma`, binary): Per-expert markings (−1=normal mapped to `False`, 1=glaucomatous mapped to `True`) and one consensus classification per image from the "Total" Excel column.
- **Localization** (`optic_disc_center`): Single keypoint from disc center text file.
- **CDR values**: Registered as raw files only; not stored in an annotation table.

### Splits
Explicit train (50 images) and test (51 images) splits from the `Training/Images/` and `Test/Images/` directories.

### Provenance
- `Drishti-GS1_diagnosis.xlsx` is registered via `process_excel`; its `raw_file_id` and `chain_id` are attached to all per-expert and consensus classification annotations.
- Each soft map PNG and boundary/localization `.txt` file is registered individually and linked to its annotation.
- CDR files are registered for provenance only; no annotations reference them as their primary source.
- Notching Excel (`notching_image_decisions.xls`) processing is disabled in the current script.

### Special processing
- Expert markings in Excel use value −1 (normal) and 1 (glaucomatous); these are converted to boolean `False`/`True` for storage in `classification_annotations`.
- Consensus column header is "Total" in the Excel file.
- Upsert order: images → segmentations → expert annotations → classifications → localizations (to satisfy foreign key constraints).
- Ingestion is idempotent due to deterministic UUID generation and upsert logic.
