# G1020

## Overview
The G1020 dataset is a retinal fundus image dataset designed for optic disc (OD) and optic cup (OC) segmentation tasks, with additional applicability to glaucoma classification. The dataset contains expert-annotated contour information for OD and OC boundaries represented as coordinate-based text files. 
The dataset includes both healthy and glaucomatous images, enabling combined segmentation and binary classification experiments related to glaucoma assessment.

## Images
- The dataset contains **1020 retinal fundus images**.
- Images are stored in **JPG** format.
- Image filenames follow the pattern `21_G1020_XXX.jpg`, where `XXX` ranges from 001 to 1020.
- Images include both healthy and glaucomatous cases.
- Image resolution, field of view, camera model, and acquisition details are not provided.
## Metadata
Metadata for the G1020 dataset is provided through a CSV file containing binary classification labels and a JSON file containing detailed per-image metadata, including segmentation annotation paths.

### Metadata Files

#### G1020.csv
This CSV file contains binary classification labels for the images.
- **imageID** -- Image identifier (e.g., `image_0.jpg`, `image_1.jpg`); note that this naming differs from the actual image filenames.
- **binaryLabels** -- Binary class label indicating health status:
- `0` -- Healthy / normal
- `1` -- Glaucomatous
 The file contains 1022 entries including the header row.

#### G1020metadata.json
This JSON file contains detailed metadata for each image in the dataset.
- **image_file_name** -- Actual image filename (e.g., `21_G1020_001.jpg`).
- **dataset_name** -- Dataset identifier, always `21_G1020`.
- **image_file_path** -- Relative path to the image file (e.g., `FundusImages/21_G1020_001.jpg`).
- **classes** -- Array containing the class label:
- `["healthy"]` -- 724 images
- `["glaucoma"]` -- 296 images
- **segmentation** -- Array of segmentation annotations associated with the image.
- **type** -- Segmentation type: `OD` (Optic Disc) or `OC` (Optic Cup).
- **contour_path** -- Relative path to the contour annotation file (e.g., `Contours/OD/21_G1020_001.txt`).
 #### Contour Annotation Files
Contour annotations are provided as plain text files in the `Contours/` directory.
- **File format** -- Plain text (`.txt`).
- **Content** -- Each line contains a space-separated `x y` coordinate pair.
- **Coordinate system** -- Pixel coordinates in the corresponding fundus image.
- **Naming convention** -- `21_G1020_XXX.txt`, where `XXX` corresponds to the image number.
- **OD contours** -- Available for all or most images.
- **OC contours** -- Available only for a subset of images.
## Splits
No explicit training, validation, or test splits are provided. The dataset contains 1020 images in total (724 healthy and 296 glaucomatous), and any data partitioning must be performed externally.

## Storage in database

### Tables populated

**`datasets`**
One record is inserted for G1020 with `modality_types=["fundus"]` and `task_types=["classification", "segmentation"]`.

**`images`**
One row per fundus image found in `Images/`. Image metadata is extracted from the physical file (dimensions, format). Fields stored: `original_image_id` (stem of the actual filename, e.g., `21_G1020_009`), `modality="fundus"`, `acquisition_date=None`, `image_quality=None`. No laterality is set (not provided by the dataset).

**Image ID derivation**: the CSV column `imageID` uses the format `image_N.jpg`; the script converts this to the actual filename by zero-padding the number to 3 digits: `image_9.jpg` → `21_G1020_009.jpg`. The `original_image_id` is the stem of that resolved filename.

**`classification_annotations`**
One binary classification per image, sourced from the `binaryLabels` column in `G1020.csv`:
- `task_type="binary"`, `class_name="glaucoma"`
- Labels: `0 → "healthy"`, `1 → "glaucoma"`
- `annotation_method="manual"`
- Provenance linked to the registered `G1020.csv` raw file.

**`segmentation_annotations`**
Up to two segmentation annotations per image — one for optic disc (OD) and one for optic cup (OC) — sourced from contour `.txt` files in `Contours/OD/` and `Contours/OC/`. Contour files are matched by the resolved image filename stem (e.g., `21_G1020_009.txt`). Each contour file contains one `x y` coordinate pair per line (line-separated format). Missing contour files are silently skipped (not all images have both OD and OC contours). `annotation_method="manual"`.

### Annotation types
- `classification` — binary glaucoma label (healthy / glaucoma) from `G1020.csv`.
- `segmentation` — optic disc and optic cup contours from `Contours/OD/` and `Contours/OC/` `.txt` files.

### Splits
No splits are created. The dataset has no explicit train/test split information and the script does not call `register_standard_splits`.

### Provenance / raw annotation files
- `G1020.csv` is registered in `raw_annotation_files` with `unified_annotation_type="classification"`. All classification annotations are linked to it via a provenance chain.
- Each individual contour `.txt` file that exists is registered separately in `raw_annotation_files` with `unified_annotation_type="segmentation"`, `file_type="txt"`. Each segmentation annotation is linked to its own contour file's provenance chain.

### Special processing
- The CSV `imageID` column uses a different naming scheme (`image_N.jpg`) from the actual files (`21_G1020_NNN.jpg`). The script parses the numeric portion and zero-pads it to 3 digits to resolve the real filename before looking up the image and contour files.
- Image dimensions are extracted from each image file (required for contour-to-mask conversion).
- Segmentation is considered optional: a row is marked successful if a classification was produced, regardless of whether contour files were found.
